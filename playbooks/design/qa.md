# Design QA Playbook

## Verification Steps
1. **Design System Compliance**: Follows existing tokens and patterns?
2. **Accessibility**: WCAG 2.1 AA — contrast ratios, ARIA labels, keyboard access
3. **Responsiveness**: Works at mobile/tablet/desktop breakpoints
4. **State Coverage**: All interactive states accounted for
5. **Consistency**: Matches the visual language of the rest of the app

## Scoring
- 0.9+: Polished, accessible, consistent
- 0.7-0.8: Functional with minor visual issues
- <0.7: Accessibility failures or design system violations
