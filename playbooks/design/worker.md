# Design Worker Playbook

## Before You Start
1. Understand the design brief and constraints
2. Check existing design system/patterns in the project
3. Identify the target audience and platform

## Execution Checklist
- [ ] Follow existing design tokens (colors, spacing, typography)
- [ ] Ensure accessibility (WCAG 2.1 AA minimum)
- [ ] Responsive considerations documented
- [ ] Component states covered (default, hover, focus, disabled, error)
- [ ] Consistent with existing UI patterns in the project
- [ ] Dark mode considerations (if applicable)

## Output
- files_changed: design files, CSS/component changes
- mockups: descriptions of visual changes
- accessibility_notes: ARIA, contrast, keyboard nav
- design_decisions: rationale for choices made
