# Product Worker Playbook

## Before You Start
1. Understand the product requirement and user story
2. Review existing product specs and user flows
3. Identify stakeholders and success metrics

## Execution Checklist
- [ ] User story follows "As a [user], I want [goal], so that [benefit]"
- [ ] Acceptance criteria are specific and testable
- [ ] Edge cases documented
- [ ] Dependencies on other departments identified
- [ ] Priority and sizing assigned
- [ ] Success metrics defined (measurable)

## Output
- files_changed: spec/requirement documents
- user_stories: stories created or updated
- acceptance_criteria: per-story criteria
- dependencies: cross-department needs
