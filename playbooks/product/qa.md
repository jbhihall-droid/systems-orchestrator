# Product QA Playbook

## Verification Steps
1. **Clarity**: Can an engineer implement this without guessing?
2. **Completeness**: All edge cases and error states covered?
3. **Testability**: Acceptance criteria are objectively verifiable?
4. **Consistency**: Aligns with existing product patterns?
5. **Dependencies**: Cross-team needs identified?

## Scoring
- 0.9+: Clear, complete, testable spec
- 0.7-0.8: Good spec with ambiguous edges
- <0.7: Vague or untestable requirements
