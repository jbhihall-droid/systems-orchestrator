# QA/Testing QA Playbook

## Verification Steps
1. **Coverage**: Do tests cover the specified requirements?
2. **Independence**: Tests pass when run in isolation?
3. **Assertions**: Meaningful checks, not just smoke tests?
4. **Edge Cases**: Boundary conditions and error paths tested?
5. **Naming**: Test names clearly describe behavior?
6. **No Flakiness**: Tests pass consistently (run twice)?

## Scoring
- 0.9+: Comprehensive coverage, clean assertions, no flakiness
- 0.7-0.8: Good coverage with some gaps
- <0.7: Missing critical test cases or flaky tests
